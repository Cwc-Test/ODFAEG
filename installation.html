Installation du framework :

Installation sur les plateformes linux.

J'ai effectué quelques vidéos qui montrent comment installer ODFAEG sur une plateforme linux.
ODFAEG (comme tout framework) utilise beaucoup de petites bibliothèques et donc il vous faudra les installer
Ne vous en faîtes pas, l'installation des ses petites biliothèque est très rapide.


Une fois les dépendances d'installée, il ne vous reste plus qu'à installer le framework.


Installation sur les plateformes windows

Elle est très similaire a celle de la plateforme linux, je vous conseille donc de regarder les vidéos ci dessus, il vous faudra juste installer le compilateur gcc ainsi openSSL autrement car sous windows il n'y a pas de dépôt apt et donc la commande apt n'existe pas! Il faut donc tout télécharger sur des sites webs.

Installer le compilateur c++.

Sous windows vous devez aller sur ce site : Ici
et télécharger un compilateur tdm-gcc 32 bits pour les plateformes 32 bits ou 64 bits pour les plateformes 64 bits. Ensuite, installer cmake-gui : Ici

Installer openSSL.

Sous windows la meilleur méthode pour installer openSSL est de télécharger MSYS : Ici
Ensuite télécharger les sources de openssl : https://www.openssl.org/ Installer ensuite perl : Ici
Lancer MSYS, aller dans le répertoire de openssl et ensuite, taper la commande suivante : perl Configure mingw Aller ensuite dans le fichier MakeFile, chercher la ligne PLATFORM et remplacer mingw par TDM-GCC-32 pour les plateformes 32 bits et TDM-GCC-64 pour les plateformes 64 bits.(Sinon ça ne compile pas) Taper ensuite les commandes suivante : mingw32-make et mingw32-make install pour installer la librairie.

Localisation du compilateur c++ dans windows avec CMake.

Lorsque vous ouvrez cmake pour compiler ODFAEG, le compilateur c++ se trouve ici : (C:/TDM-GCC-32/bin ou bien C:/TDM-GCC-64/bin)

Attention

Pour l'instant, seul le "linkage" du framework en mode statique fonctionne. Vous devez donc "linker" toutes les dépendances de ODFAEG lors de la compilation de votre projet.
